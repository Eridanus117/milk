# 回归清单

发布前至少覆盖以下检查。

## 首页与导航

- 默认进入首页会话列表，而不是直接进入某个聊天
- 顶部 `功能 / 首页 / 设置` 三个按钮在桌面端和移动端都保持横排可读
- 点击 `首页` 能从任意聊天返回会话列表
- 点击 `功能` 能打开功能列表，不被其他遮罩拦截
- 点击 `当前设置` 能打开当前聊天设置，自定义会话会给出明确提示

## 系统 6 聊

- 首页存在 `1` 个群聊和 `5` 个角色单聊入口
- 切到任一角色单聊，头部名称与头像正确
- 角色单聊聊天记录彼此独立
- 群聊最近消息、未读红点、时间展示正常

## 回复正确性

- 单聊即时回复不会串到其他角色字卡
- 群聊发言人与字卡池匹配，不会出现“人对但卡错”或“卡对但人错”
- 前台即时回复与后台离屏回复在同一角色上表现一致

## 后台消息设置

- `恢复` 开关点击后有即时视觉反馈
- `主动发消息` 开关点击后有即时视觉反馈
- 关闭 `主动发消息` 后，非当前会话不再离屏新增消息
- 开启 `恢复` 后，离开再回来会补发
- 当前会话恢复补发时不增加未读
- 非当前会话离屏新增消息时会增加未读

## 版本更新

- 设置页能看到构建版本、构建时间
- `检查更新` 能识别远端较新 `assetVersion`
- 检测到新版本后按钮变成 `立即更新`
- `立即更新` 会先保存，再刷新切到新版
- 刷新后版本信息更新为新值，状态显示“当前已是最新版本”

## 外观与主题

- 外观页可见 `日用 / 夜用` 切换入口
- `日用 / 夜用` 点击后 `data-theme` 正确变化
- 配色切换与明暗切换可叠加，不互相覆盖

## 移动端

- 390px 左右宽度下首页顶部按钮不竖排
- 首页会话列表首屏可正常浏览
- 功能列表、当前聊天设置弹层不溢出视口
- 输入区、聊天区、首页区之间切换没有明显错位
